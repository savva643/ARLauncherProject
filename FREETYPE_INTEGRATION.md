# Интеграция FreeType для нормальных шрифтов

## Что было сделано:

1. **Добавлен FreeType в CMakeLists.txt**
   - Поиск библиотеки FreeType через find_package или pkg-config
   - Автоматическое определение путей для Linux и Windows
   - Добавлен USE_FREETYPE макрос для условной компиляции

2. **Создан класс FontRenderer**
   - `ARLauncher/include/FontRenderer.h` - заголовочный файл
   - `ARLauncher/src/FontRenderer.cpp` - реализация
   - Поддержка загрузки TTF/OTF шрифтов
   - Кэширование глифов в текстурах OpenGL
   - Поддержка UTF-8 (включая русские символы)
   - Рендеринг текста с правильным позиционированием

3. **Обновлен Text.cpp**
   - Использует FontRenderer вместо простых точек
   - Fallback на простые точки если FreeType не доступен
   - Поддержка масштабирования и цвета

4. **Интеграция в Application**
   - FontRenderer инициализируется в `initializeUI()`
   - Автоматический поиск системных шрифтов
   - Установка глобального FontRenderer для всех Text элементов

## Установка FreeType:

### Linux:
```bash
sudo apt-get install libfreetype6-dev
```

### Windows (vcpkg):
```bash
vcpkg install freetype:x64-windows
```

### macOS:
```bash
brew install freetype
```

## Использование:

После установки FreeType, проект автоматически найдет библиотеку и будет использовать нормальные шрифты вместо простых глифов.

Если FreeType не найден, приложение будет работать с fallback на простые точки (как было раньше).

## Преимущества:

✅ Поддержка любых TTF/OTF шрифтов
✅ Поддержка UTF-8 (русские, китайские, арабские символы и т.д.)
✅ Красивое отображение текста
✅ Правильное кернинг и метрики шрифтов
✅ Масштабирование без потери качества
